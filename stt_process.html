STT (Speech-to-Text) 처리 프로세스

Supervisor Agent 음성 입력 → 텍스트 변환 → execute_task 연동 전체 흐름

프로젝트: smart-supervisor-sub-agent 작성 기준: v1.0 STT Provider: OpenAI Realtime / Google Gemini Live

1. 엔드포인트 구조

단순 채팅과 음성은 서로 다른 엔드포인트를 사용한다. 이유는 프로토콜과 입력 타입이 근본적으로 다르기 때문.

구분 엔드포인트 프로토콜 입력 타입 파일
채팅 POST /a2a/supervisor HTTP (JSON-RPC 2.0) JSON text supervisor.py:36
음성 WS /a2a/supervisor/voice/stream WebSocket PCM16 binary + JSON supervisor.py:181
왜 분리했는가? 음성은 실시간 오디오 청크 스트리밍이 필요하다. PCM16 청크를 HTTP로 base64 인코딩해서 보내면 chunk 단위 처리 불가, 전체 업로드 후 처리 → 레이턴시 폭발. WebSocket은 한 번의 핸드셰이크로 지속적인 양방향 스트리밍이 가능하다.

2. 전체 아키텍처 흐름

  1. Browser → Server: WebSocket connect (/voice/stream?session_id=xxx)
  2. Server: adapter 생성 (VoiceAdapterFactory) + adapter.connect() → STT Provider WS 연결
  3. receive_loop: Browser PCM16 binary → adapter.send_audio(bytes) → STT Provider (base64 JSON)
  4. send_loop: adapter.listen() → STT Provider 이벤트 스트림 구독
  5. Partial {"type":"transcript"} → Browser 실시간 자막 (UX 전용)
  6. final_transcript → agent_service.execute_task(session_id, transcript, request_id) (채팅과 동일)
핵심 설계: 음성은 순수한 입력 수단. STT로 텍스트 변환 후 채팅과 동일한 execute_task(). 별도 분기/플래그 없음.

3.5 execute_task 연결

# 채팅 진입 (supervisor.py:59)
result = await agent_service.execute_task(params.session_id, params.message, params.request_id)

# 음성 진입 (supervisor.py:236)
result = await agent_service.execute_task(session_id, transcript, request_id)
시그니처 완전 동일. message와 transcript 모두 str. 이후 HITL 평가, 그래프 실행, 하위 에이전트 라우팅 등 모든 로직을 공유한다.

4. VAD (Voice Activity Detection) 설정

OpenAI Realtime Adapter의 _update_session()에서 VAD 파라미터가 설정된다.

8. 채팅과 음성 비교 요약

채팅 (POST /a2a/supervisor)

HTTP JSON-RPC. message/send 메서드.
요청: {"jsonrpc":"2.0","method":"message/send","params":{"session_id":"...","message":"..."}}
응답: JSON-RPC Response
프론트: app.js

음성 (WS /a2a/supervisor/voice/stream)

WebSocket. PCM16 binary.
Browser → Server: audio chunks (binary) + commit (JSON)
Server → Browser: transcript events (JSON)
프론트: voice.js
STT 완료 후 execute_task() 자동 트리거

공통

execute_task(session_id, text, request_id)
→ HITL 평가
→ Supervisor Graph 실행
→ 하위 에이전트 라우팅
→ SSE 이벤트 스트리밍

차이

입력 방식 (text vs audio).
STT 레이어 유무.
프로토콜 (HTTP vs WS).
실시간성 (음성은 chunk streaming).
