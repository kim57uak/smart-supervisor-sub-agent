시스템 개요 (System Overview)

본 프로젝트는 Supervisor Agent와 Sub-Agent로 구성된 A2A(Agent-to-Agent) 기반 멀티 에이전트 시스템이다. Supervisor는 사용자 요청을 분석하고 적합한 Sub-Agent에 작업을 라우팅하며, Sub-Agent는 MCP(Model Context Protocol) 도구를 통해 실제 비즈니스 로직을 수행한다.

음성 입력(STT)과 텍스트 채팅을 모두 지원하며, 두 입력 모두 동일한 execute_task() 실행 파이프라인을 통해 처리된다. 음성은 OpenAI Realtime API 또는 Google Gemini Live API를 통해 실시간 전사(transcription)된 후 Supervisor 코어 로직으로 전달된다.

핵심 설계 원칙: 음성은 순수한 입력 수단. STT로 텍스트 변환 후 채팅과 동일한 execute_task() 로직. 별도 분기/플래그 없음.

Supervisor Agent (마스터 오케스트레이터)

Supervisor는 시스템의 중앙 제어 장치다. 모든 사용자 요청은 Supervisor를 통해 들어오며, 요청 분석 → 실행 계획 수립 → HITL 평가 → Sub-Agent 호출 → 결과 합성의 전 과정을 주관한다.

음성 통합 (Voice Pattern 3)

  • Server-Side Trigger: 음성 인식 완료 후 브라우저를 거치지 않고 서버에서 즉시 에이전트 작업 실행
  • STT Provider: OpenAI Realtime API (Whisper) 또는 Google Gemini Live API 선택 가능
  • Voice Adapter: 공통 VoiceAdapter 인터페이스로 Provider 전환 용이 (VoiceAdapterFactory)
  • WebSocket: PCM16 오디오 청크 실시간 스트리밍 + VAD(Voice Activity Detection) 기반 발화 종료 감지
  • VAD 설정: threshold=0.4, prefix_padding_ms=600, silence_duration_ms=800

Sub-Agent (실행 에이전트)

Sub-Agent는 Supervisor로부터 요청을 받아 실제 업무를 수행하는 실행형 에이전트다. MCP(Model Context Protocol)를 통해 외부 도구와 연동하며, 내부적으로 Producer-Consumer 패턴으로 비동기 처리된다.

실행 흐름 (Phase 1 — Producer)

  1. a2a_handler.py가 Supervisor의 POST /a2a 요청 수신
  2. authorization_service.py — scope 기반 권한 검증
  3. task_queue.py — 실행 컨텍스트를 Redis Queue에 인큐
  4. HTTP 202 Accepted + task_id 즉시 반환 (비동기 전환)

실행 흐름 (Phase 2 — Consumer)

  1. worker.py — Redis BRPOP/XREAD로 태스크 지속 폴링
  2. LangGraphFactory — 도구 선택 → LLM ToolPlan 수립
  3. McpInfrastructure — MCP Handshake → 도구 실행
  4. AgentProgressPublisher — 진행 상황 Redis Stream 발행
  5. LlmComposer — 최종 답변 생성 및 스트리밍

시작하기 (Quick Start)

1. 가상환경 및 의존성 구축

python3 -m venv .venv
source .venv/bin/activate
pip install -r src/supervisor-agent/requirements.txt

2. 환경 변수 설정

# .env 파일 생성
OPENAI_API_KEY=sk-...
GOOGLE_API_KEY=...
REDIS_URL=redis://localhost:6379/0

# voice provider 선택
VOICE_PROVIDER=openai    # 또는 google

3. 서비스 기동

./start_all.sh

4. 기능 검증

python verify_session_id_e2e.py
